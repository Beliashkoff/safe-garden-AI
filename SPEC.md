# SPEC — Safe Garden AI

Продуктовая спецификация мобильного приложения для диагностики проблем растений с помощью Claude Opus и нативной рекомендации удобрений компании.

> Этот документ — **что и зачем мы строим**. Технические детали в `ARCHITECTURE.md`, последовательность работ в `ROADMAP.md`.

---

## 1. Цель и пользователи

### 1.1 Проблема
Садовод/огородник видит, что с растением что-то не так (увядает, желтеет, есть пятна, плохо плодоносит), но не знает причины и не уверен, какое удобрение/лечение применять.

### 1.2 Решение
Один экран — чат с ИИ, в который пользователь:
- присылает фото проблемного растения,
- может голосом описать, что произошло,
- получает структурированный ответ: диагноз → причина → пошаговое лечение → конкретное удобрение из каталога компании.

### 1.3 Целевая аудитория
- Дачники и владельцы огородов в России и СНГ
- Возраст: 30–65, преимущественно мобильное использование
- Уровень техники: средний/низкий — UX должен быть максимально простым

### 1.4 Метрика успеха
- Активация: % новых пользователей, отправивших ≥ 1 фото в первой сессии
- Удержание: D7, D30 retention
- Бизнес: % сессий, в которых пользователь увидел рекомендацию удобрения и совершил тап по карточке

## 2. Функциональные требования

| ID  | Фича                                          | Критерий приёмки (DoD)                                                                                                                                                           |
| --- | --------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| F1  | Авторизация при первом входе                  | Apple Sign-In (iOS), Google Sign-In (iOS+Android), Email + 6-значный OTP. Токены хранятся в `flutter_secure_storage`. Повторный вход — автоматический по refresh-токену.         |
| F2  | Один экран — чат с ИИ                         | После авторизации сразу попадаем в чат. Один `conversation` на пользователя, история подгружается с сервера, синхронизируется между устройствами.                                |
| F3  | Отправка текстового сообщения                 | До 4000 символов. Стримящийся ответ в реальном времени.                                                                                                                          |
| F4  | Отправка фото                                 | Камера или галерея, 1–4 фото за сообщение. Сжатие до max 1920×1080, JPEG q=85. Загрузка через presigned URL напрямую в Object Storage.                                          |
| F5  | Отправка голосового сообщения                 | Запись с индикатором длительности (max 60s), отмена. Загрузка → транскрибация на сервере → отправка как текст с пометкой `voice`.                                                |
| F6  | Стриминг ответа Claude                        | Server-Sent Events. Первый токен ≤ 3s p95. Возможность отмены (cancel) на клиенте, сервер прерывает запрос к Claude.                                                             |
| F7  | Структурированный ответ                       | Маркдаун-секции: «Что вижу» → «Возможные причины» → «Что делать (шаги)» → карточка-рекомендация удобрения.                                                                       |
| F8  | Карточка удобрения                            | Нативный UI-блок: фото, название, краткое описание, deeplink/ссылка. Подбирается Claude через tool call `recommend_fertilizer` из внутреннего каталога.                         |
| F9  | Удаление аккаунта и всех данных               | Доступно из настроек. Каскадное удаление: записи в БД, объекты в Object Storage, отзыв OAuth refresh-токенов. Требование App Store §5.1.1(v) и Google Play.                      |
| F10 | Локализация                                   | v1 — русский. Архитектура готова к мультиязычности (`intl` в Flutter, `Accept-Language` на бэкенде).                                                                            |

## 3. Нефункциональные требования

- **Платформы:** iOS 14+, Android 8+ (API level 26+).
- **Производительность:**
  - TTFB (time to first byte) ответа Claude ≤ 3s p95.
  - Холодный старт приложения ≤ 2s p95.
  - Загрузка фото 2 МБ ≤ 5s на 4G.
- **Доступность:** базовая поддержка VoiceOver (iOS) / TalkBack (Android), контраст AA.
- **Офлайн:** показ истории из локального кэша; отправка блокируется с понятным сообщением.
- **Безопасность данных:** TLS 1.2+, шифрование at-rest, секреты в KMS. Подробности в `ARCHITECTURE.md` §7.
- **Юридическое соответствие:**
  - 152-ФЗ (хранение ПДн в РФ).
  - Apple App Store Review Guidelines (§5.1.1, §5.1.4 Sign in with Apple обязателен при наличии стороннего OAuth).
  - Google Play Data Safety + Account Deletion Policy.
- **Наблюдаемость:** структурированные логи (slog/JSON), метрики (Prometheus-формат), трассировка ключевых операций, Sentry для мобильного и бэкенда.

## 4. Границы

### 4.1 Делаем в v1
- Один чат на пользователя
- Текст / фото / голос
- Авторизация Apple / Google / Email
- Каталог удобрений + рекомендации через tool use
- Удаление аккаунта
- Стриминг ответа
- Русская локализация
- Деплой в Yandex Cloud (РФ-юрисдикция)
- Публикация в App Store и Google Play

### 4.2 НЕ делаем в v1 (отложено)
- Несколько отдельных чатов / тредов
- Социальные функции (комментарии, шейринг, лайки)
- Покупки в приложении, подписки
- Веб-версия
- Watch / iPad-оптимизация (минимально работает на iPad в режиме iPhone)
- Push-уведомления (планируется в v2)
- RAG / собственная база знаний с pgvector (v2 — после валидации MVP-качества ответов Claude)

### 4.3 Спросить пользователя перед
- Изменением схемы БД в продакшене (миграции `down`)
- Добавлением сторонних SDK (особенно трекеры/аналитика)
- Сменой провайдера LLM или модели Claude
- Изменением политики данных или ToS
- Изменением домена / SSL / store-листинга
- Любыми операциями `git push --force`, `git reset --hard origin`

### 4.4 Никогда
- Логировать содержимое сообщений / фото / аудио в открытые логи или внешние системы
- Хранить OAuth-токены, refresh-токены, OTP-коды в plaintext
- Передавать медиа-файлы без TLS
- Отдавать сообщения чужого пользователя без проверки `user_id` владельца
- Отключать проверку email при повторных входах с того же провайдера
- Хардкодить ключи API в коде или коммитах (в т.ч. тестовые)
- Использовать `--no-verify` при коммите/пуше

## 6. Стратегия тестирования

### 6.1 Backend
- **Unit:** `testify`, табличные тесты, моки только на внешние границы (Anthropic API, S3, SMTP).
- **Integration:** Postgres через `testcontainers-go`, реальные миграции, фикстуры.
- **E2E (smoke):** httptest-сервер + реальный Claude в стейджинг-окружении, прогон на CI 1×/день.
- **Покрытие:** ≥ 70% по `internal/usecase` и `internal/transport`.

### 6.2 Mobile
- **Unit:** логика репозиториев и нотифаеров.
- **Widget:** ключевые экраны и компоненты.
- **Golden tests:** карточка удобрения, сообщение пользователя, сообщение ИИ.
- **integration_test:** сценарий «логин → отправка фото → получение ответа» против моков.

### 6.3 Ручное тестирование перед релизом
- Чек-лист на устройствах: iPhone (старая модель iOS 14), iPhone (актуальная), Android (старый, новый).
- Сценарии: разрешения камеры/микрофона/фото, плохая сеть, отмена записи, длинный ответ, вход после переустановки.

## 7. Открытые вопросы (статус на 2026-05-08)

| #   | Вопрос                                                                | Статус        | Решение / комментарий                                                                                                          |
| --- | --------------------------------------------------------------------- | ------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| Q1  | Доступ к Anthropic API из инфраструктуры в РФ                         | **Закрыт**    | Через **отдельный LLM-worker на HostKey Frankfurt** (`vm.v2-nano`, оплата ₽). Anthropic-аккаунт оформлен на иностранное юрлицо/карту (Anthropic в рублях не принимает). См. ARCH §11 и §11.7 — принятые риски с РФ-юрлицом провайдера. AWS Bedrock и прямой вызов из РФ — не работают. |
| Q2  | Каталог удобрений: где хранится, нужна ли админка                     | Отложен       | Каталог поступит позже. До этого `fertilizers` пуст, tool возвращает «нет подходящего». Архитектура (таблица + tool) делается на Этапе 5. |
| Q3  | Email-провайдер для OTP                                               | **Закрыт**    | **Yandex 360 SMTP** (`noreply@<domain>`). Рублёвая оплата, нативный для Yandex-стэка.                                          |
| Q4  | Транскрипция голоса                                                   | **Закрыт**    | **Yandex SpeechKit v3 (sync recognize)**. Рублёвая оплата, лучшее качество для русского, без экспортных рисков. m4a→OggOpus на бэке (`ffmpeg`). |
| Q5  | Дизайн UI                                                             | **Закрыт**    | Делаем своими силами. Заложить ~3–5 дней дизайна перед каждым этапом, начиная с Этапа 1.                                       |
| Q6  | Privacy Policy / Terms of Service (RU + EN)                           | Отложен       | Готовится к Этапу 7. До этого приложение не сабмитим в сторы.                                                                  |
| Q7  | Apple Developer Account и Google Play Console                         | Отложен       | Заказчик оформит самостоятельно в начале Этапа 7.                                                                              |

## 8. Принятые решения

| #   | Решение                                                                                  | Причина                                                                |
| --- | ---------------------------------------------------------------------------------------- | ---------------------------------------------------------------------- |
| D1  | Frontend: **Flutter**                                                                    | Единая кодовая база, лучше для медиа-тяжёлого UI, проще нативные плагины |
| D2  | Backend: **Go** (chi + pgx + sqlc)                                                       | Указано заказчиком; типобезопасный SQL, быстрая разработка             |
| D3  | LLM: **Claude Opus 4.x** через `anthropic-sdk-go` (на стороне LLM-worker'а вне РФ)       | Указано заказчиком                                                     |
| D4  | DB: **PostgreSQL**                                                                       | Зрелое решение; pgvector для будущего RAG                              |
| D5  | Object Storage: **S3-совместимое** (Yandex Object Storage)                               | РФ-юрисдикция, S3 API совместимость                                    |
| D6  | Облако: **Yandex Cloud**                                                                 | 152-ФЗ, низкая латентность для РФ-пользователей                        |
| D7  | RAG отложен до v2                                                                        | Сначала валидируем качество ответов Claude без БЗ                      |
| D8  | Один чат на пользователя                                                                 | Указано заказчиком; простой UX                                         |
| D9  | Стриминг через **SSE** (не WebSocket)                                                    | Проще на мобильном (HTTP/2), достаточно для односторонних потоков      |
| D10 | Refresh-токены **opaque** (не JWT), хэшируются в БД                                      | Возможность отзыва, защита от утечки                                   |
| D11 | LLM-доступ через **split LLM-worker** на HostKey Frankfurt (`vm.v2-nano`, ~830 ₽/мес, ООО «АЙТИБ») | Anthropic блокирует все российские AS на уровне политики и Cloudflare; AWS Bedrock не работает для РФ-инфраструктуры; worker даёт чистый Frankfurt-IP, оплата ₽, изолирует PII (152-ФЗ). Юрлицо провайдера РФ — принятые риски и план миграции на иностранного провайдера зафиксированы в ARCH §11.7. |
| D12 | Email-провайдер: **Yandex 360 SMTP** (`noreply@<domain>`)                                | РФ-юрлицо, рублёвая оплата, нативный для Yandex-стэка.                 |
| D13 | Транскрипция: **Yandex SpeechKit v3 (sync recognize)**, m4a→OggOpus конвертация (ffmpeg) | Лучшее качество для русского, оплата ₽, без экспортных рисков.         |
| D14 | Дизайн делаем своими силами (без отдельного дизайнера в v1)                              | Бюджетная модель v1.                                                   |
| D15 | Окружения: только **dev (локально) + prod**. **Stage не планируется** даже после релиза. | Минимизация инфраструктуры и стоимости. До публикации в сторы prod используется и для ручного тестирования. Если в будущем понадобится отдельная тестовая инфра — оформим как отдельное продуктовое решение. |
| D16 | Деплой: **Docker Compose на VM** (не Kubernetes)                                         | Просто, дёшево, легко дебажить. Миграция на K8s — после роста нагрузки. |
| D17 | Секреты на worker'е: **.env на LUKS-зашифрованном томе** внутри VM, права 600            | Один VPS, не нужен отдельный Vault. Шифрование тома защищает от snapshot/IPMI-доступа оператора провайдера. |
